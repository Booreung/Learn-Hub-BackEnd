1. 커밋 메시지
1) 메시지 타입
- 추가 : 새로운 기능 또는 파일 추가
- 수정 : 코드나 기능 수정
- 버그 : 버그 수정
- 리팩터 : 코드 리팩토링
- 테스트 : 테스트 코드 작성
- 문서 : 문서 수정
- 환경설정 : 빌드 시스템 또는 외부 의존성에 관한 설정 변경
- 성능 : 성능 개선
- 배포 : 배포 관련 커밋
- 보안 : 보안 관련 수정
- 일단 완성
- 완성

2) 예시
"추가: 사용자 로그인 기능 추가

- 로그인 페이지 UI 구현
- 로그인 API 연동"

"버그: 로그인 시 비밀번호 검증 오류 수정

- 비밀번호 검증 로직 수정
- 관련 테스트 코드 추가"

2. 브랜치명
영문이름이니셜 + "-" + 기능
ex : [JSJ]-회원가입

- 기능 구현 시작 시 브랜치 생성
- 기능 구현 완료 후 해당 브랜치 삭제

3. 변경 사항
개발하시다가 요구사항 바꿔야 할 거 같거나 더 추가해야 할 거 생길 겁니다. 바로바로 말씀해주세요.

4. DB 변경 사항
DB 변경사항, 특히 엔티티 클래스 수정은 팀장에게 꼭 문의해주세요.
